# 🚀 快速入门指南

## 5分钟快速上手

### 第一步：安装依赖

```bash
pip install flask flask-cors dashscope playwright
playwright install chromium
```

### 第二步：设置 API Key

**Windows (PowerShell):**
```powershell
$env:DASHSCOPE_API_KEY = "sk-your-api-key"
```

**Windows (CMD):**
```cmd
set DASHSCOPE_API_KEY=sk-your-api-key
```

**Linux/Mac:**
```bash
export DASHSCOPE_API_KEY="sk-your-api-key"
```

### 第三步：启动服务

**方式1: 使用启动脚本（推荐）**

Windows:
```bash
start_server.bat
```

Linux/Mac:
```bash
chmod +x start_server.sh
./start_server.sh
```

**方式2: 直接运行**
```bash
python main.py
```

### 第四步：访问网页

打开浏览器，访问：**http://localhost:5000**

### 第五步：开始使用

1. **上传图片**: 拖拽或点击上传参考图表
2. **配置参数**（可选）:
   - 最大迭代轮数: 1-10（默认5）
   - 验证阈值: 0-1（默认0.75）
3. **点击"开始任务"**
4. **观察进度**: 实时查看各 Agent 工作状态
5. **获取结果**: 下载代码、图片或在线预览

## 💡 使用技巧

### 1. 理解状态指示

- **黄色**: 等待中
- **蓝色**: 运行中
- **绿色**: 已完成
- **红色**: 失败/错误

### 2. Agent 工作流程

```
Agent1 (代码生成) 
    ↓
渲染截图
    ↓
Agent2 (视觉评判) 
    ↓
Agent3 (代码评估) 
    ↓
Agent4 (反馈修订) 
    ↓
验证器
    ↓
通过？→ 完成
未通过？→ 返回 Agent2 继续迭代
```

### 3. 参数调优建议

**快速测试**:
- max_loops: 3
- threshold: 0.7

**高质量输出**:
- max_loops: 8
- threshold: 0.85

**平衡模式**（默认）:
- max_loops: 5
- threshold: 0.75

### 4. 多任务管理

- 可以同时上传多个图片创建多个任务
- 在"历史任务"列表中切换查看
- 每个任务独立运行，互不干扰

## ❓ 常见问题

### Q1: 页面无法访问？
**A**: 检查：
1. 服务是否成功启动（终端应有日志输出）
2. 端口 5000 是否被占用
3. 防火墙是否阻止

### Q2: 任务一直卡在"等待中"？
**A**: 检查：
1. API Key 是否正确设置
2. 网络连接是否正常
3. 查看后端终端的错误信息

### Q3: 生成的图片质量不好？
**A**: 尝试：
1. 提高 threshold（如 0.85）
2. 增加 max_loops（如 8）
3. 确保原图清晰度高

### Q4: 如何查看完整代码？
**A**: 
1. 点击"下载代码"按钮
2. 或在 outputs/{task_id}/ 目录查找

### Q5: 任务完成后如何重新查看？
**A**: 
1. 刷新页面
2. 在"历史任务"列表中点击对应任务
3. 或直接访问任务 ID

## 🎯 最佳实践

### 1. 图片准备
- ✅ 使用清晰的图表截图
- ✅ 分辨率建议 800x600 以上
- ✅ 避免模糊或压缩过度
- ❌ 不要包含无关元素

### 2. 参数选择
- 首次使用保持默认值
- 根据结果逐步调整
- 记录成功的参数组合

### 3. 结果验证
- 下载 HTML 在浏览器中交互查看
- 对比原图和生成图的细节
- 检查数据和标签准确性

### 4. 资源管理
- 定期清理 uploads/ 和 outputs/ 目录
- 避免同时运行过多任务
- 长时间不用时关闭服务
