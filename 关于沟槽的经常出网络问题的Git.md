### 🛠️ Git 冲突解决与合并全流程总结

当你执行 `git pull` 或 `git merge` 时，如果 Git 提示冲突（`CONFLICT`），请按照以下 4 个步骤操作：

#### 第一步：拉取代码
在命令行中执行拉取命令。如果是因为两个独立的项目合并（如之前的 `unrelated histories` 错误），记得加上参数。

```bash
git pull origin <远程分支名> --allow-unrelated-histories
```
- **结果**：Git 会尝试合并，如果失败，会提示哪些文件有冲突（例如 `README.md`）。

#### 第二步：手动修复冲突文件
这是最关键的一步，需要你人工介入。

1.  **打开文件**：在项目文件夹中找到 Git 提示冲突的文件。
2.  **寻找标记**：文件中会出现类似下面的奇怪符号：
    ```text
    <<<<<<< HEAD
    这里是你本地的代码内容
    =======
    这里是远程仓库的代码内容
    >>>>>>> 远程分支名
    ```
3.  **编辑**：
    - **保留想要的**：决定是保留上面的代码、下面的代码，还是两边都保留。
    - **删除标记**：**必须**删除 `<<<<<<<`、`=======` 和 `>>>>>>>` 这些符号，只保留干净的代码。
4.  **保存**：保存并关闭文件。

#### 第三步：标记为“已解决”
回到命令行，告诉 Git 你已经修好了文件。

```bash
git add .
```
- **作用**：这将把修改后的文件标记为“冲突已解决，准备合并”。

#### 第四步：完成合并提交
执行提交命令，正式完成这次合并操作。

```bash
git commit
```
- **注意**：这通常会打开一个编辑器（Vim）让你输入合并说明。
    - **保存退出方法**：按 `i` 进入编辑模式（可选修改文字），按 `Esc` 退出编辑模式，输入 `:wq` 并回车保存退出。

#### 第五步：推送到远程
合并完成后，你的本地代码就和远程同步了，现在可以推送了。

```bash
git push origin <本地分支>:<远程分支>
```
- 例如：`git push origin master:HuaiTaov3.0(WebUI)`

---

### 💡 核心口诀
> **先拉取，修文件，删标记，Add 它，Commit 它，最后 Push。**
