# ECharts 图表代码评判报告

> 本报告基于视觉 Agent 提供的《图表评估报告》，结合 ECharts 官方文档与最佳实践，对所提交的内联 JavaScript 绘图代码进行**专业级可维护性与表现一致性评审**。重点关注：**是否能稳定复现参考图的视觉细节**，而非仅实现基础功能。

---

## ✅ 一、可读性（Readability）

| 维度 | 评价 | 说明 |
|------|------|------|
| **命名规范** | ⚠️ 中等偏下 | `dataCD`, `dataBD` 等变量名符合语义，但 `randomNormal()`、`generateData()` 等辅助函数缺乏 JSDoc 注释；`xMean/xStd` 等参数未说明物理含义（如 “Centroid offset 均值/标准差”），不利于跨团队协作。 |
| **注释质量** | ⚠️ 不足 | 仅有少量中文注释（如 `// CD: 左上 (x<0, y>0)`），关键逻辑（如 `if (x > -60 && x < 110...)` 的裁剪边界依据）、坐标轴 formatter 的字母标注（`-50(C)`）未解释其来源（是否对应原始论文分组标签？），**缺乏“为什么这么写”的上下文说明**。 |
| **结构清晰度** | ✅ 良好 | 数据生成、颜色定义、配置组装三段分离明确；`graphic` 组件中统计框使用 `group` + 多子元素组织合理，层次清晰。 |

✅ **改进建议**：  
- 为所有函数添加 JSDoc，注明参数含义、返回值及业务约束（如 `@param {number} xMean - Centroid offset 均值（单位：mm），取值范围应覆盖 [-20, 40]`）；  
- 在 `axisLabel.formatter` 注释中说明 `(C)/(B)/(A)` 等字母是原始图中标注的实验条件代号，非随意添加；  
- 将 `colors` 对象移至顶部常量区，并添加注释说明色值来源（如 `// 参考图中 CD 组实际 HEX: #8FD3F4`）。

## ✅ 二、模块化（Modularity）

| 维度 | 评价 | 说明 |
|------|------|------|
| **数据与配置分离** | ⚠️ 部分耦合 | 数据生成逻辑（`generateData`）与图表配置 `option` 混在同一作用域，且 `dataCD` 等变量直接嵌入 `series.data`，**无法独立测试或替换数据源**（如换成真实 CSV 加载）。 |
| **配置可复用性** | ❌ 较弱 | `option` 是硬编码对象，未封装为函数（如 `getChartOption(dataMap)`），`graphic` 中统计框坐标（`left: '65%', bottom: '18%'`）为魔法数值，**无响应式或容器适配逻辑**，缩放时易错位。 |
| **主题/样式抽象** | ❌ 缺失 | 字体（`KaiTi`）、字号（`14px`/`16px`）、颜色（`#333`）、边框（`#999`）等样式散落在各处，未提取为统一主题对象（如 `theme = { fontSize: 14, textColor: '#333', ... }`）。 |

✅ **改进建议**：  
- 将数据生成逻辑抽离为独立模块（如 `dataGenerator.js`），暴露 `generateAllGroups()` 接口；  
- 封装 `createScatterOption(dataMap, config = {})` 函数，接收数据映射 `{CD: [...], BD: [...]}` 和可选配置（如 `showTitle: false`, `legendCols: 2`）；  
- 提取 `CHART_THEME` 常量对象，统一管理字体、颜色、间距等；  
- `graphic` 中统计框位置改用 `right: 40, bottom: 40`（像素绝对定位）替代百分比，避免响应式错位。

## ✅ 三、可测试性（Testability）

| 维度 | 评价 | 说明 |
|------|------|------|
| **数据可注入性** | ⚠️ 有限 | `generateData()` 是纯函数，理论上可单元测试，但当前代码中它被立即调用并赋值给 `dataCD` 等变量，**未提供 mock 入口**；若需验证某组数据分布，必须 patch `Math.random`，成本高。 |
| **配置可断言性** | ✅ 良好 | `option` 是 plain object，可直接 `JSON.stringify(option.xAxis)` 进行快照测试；`markLine`、`grid` 等关键配置项路径明确，适合做深度 equal 断言。 |
| **视觉回归可行性** | ❌ 缺失 | 无任何自动化视觉测试钩子（如 Puppeteer 截图比对），依赖人工肉眼验收，**无法保障后续修改不破坏视觉一致性**。 |

✅ **改进建议**：  
- 将 `generateData` 改为接受 `rng` 参数（随机数生成器），便于注入 `seedrandom` 实现确定性测试；  
- 导出 `option` 配置对象（如 `export const baseOption = {...}`），支持 Jest 测试其结构（如 `expect(option.xAxis.min).toBe(-50)`）；  
- 在 CI 中增加 Puppeteer 截图 + Pixelmatch 差异检测（需配套 Docker 环境）。

## 🚨 四、与视觉报告的一致性（Critical Gap Analysis）

> 🔑 **核心结论：当前代码无法复现参考图的关键视觉特征，主因是 ECharts 默认行为与 Matplotlib 默认行为差异未被主动收敛，且部分配置项被错误忽略或误用。**

| 视觉差异点 | 代码现状 | 根本原因 | **具体改进指导（必须执行）** |
|------------|----------|-----------|-----------------------------|
| **X/Y 轴刻度不精确** | `xAxis.min/max` 和 `yAxis.min/max` 已设，但未强制 `interval` 或 `splitNumber`，ECharts 自动计算导致刻度杂乱（如 `-30, 30, 60...`） | ECharts 的 `min/max` 仅控制范围，**不控制刻度密度**；需显式设置 `interval` 或 `data` 数组 | ✅ **X轴**：在 `xAxis` 中添加 `interval: 50`，并确保 `data: [-50, 0, 50, 100]`（覆盖全部目标刻度）<br>✅ **Y轴**：在 `yAxis` 中添加 `interval: 2`，并设置 `data: Array.from({length: 10}, (_, i) => -8 + i*2)`（即 `[-8,-6,...,10]`） |
| **顶部/右侧多余坐标轴** | 代码中无 `top/right` spine 设置，但视觉报告指出存在“顶部短坐标轴” | ECharts 默认显示 `top` 和 `right` 的 `axisLine`（即使 `show: false` 未显式声明） | ✅ 在 `xAxis` 和 `yAxis` 中**显式关闭**：<br>`axisLine: { show: true, onZero: false }` → 改为 `axisLine: { show: true, onZero: true }`（仅保留过零线）<br>`splitLine: { show: false }` → **补充** `axisTick: { show: false }`, `axisLabel: { show: true }`，并**在 `xAxis` 中添加 `position: 'bottom'`，在 `yAxis` 中添加 `position: 'left'`**（强制仅渲染底/左轴） |
| **图例单列无边框** | `orient: 'vertical'` + `left/top` 定位，但未设 `borderWidth`/`borderColor` | `legend` 缺少 `borderWidth: 1`, `borderColor: '#ccc'`, `backgroundColor: 'rgba(255,255,255,0.9)'`；`ncol=2` 未设置 | ✅ 将 `legend` 配置改为：<br>`{ orient: 'horizontal', left: 'center', top: '5%', itemWidth: 12, itemHeight: 12, borderWidth: 1, borderColor: '#999', backgroundColor: 'rgba(255,255,255,0.95)', ncol: 2 }`<br>⚠️ 注意：`orient: 'horizontal'` + `ncol: 2` 才能实现双列，`vertical` 下 `ncol` 无效！ |
| **标题栏无青色背景条** | `title.text` 仅为文字，无背景矩形 | `title` 组件不支持背景色，必须用 `graphic` 绘制顶部色块 | ✅ **新增 graphic 元素**（置于最顶层）：<br>`{ type: 'rect', z: 100, left: 0, top: 0, shape: { width: '100%', height: 40 }, style: { fill: '#00CED1' /* 青色 */ } }`<br>✅ 同时将原 `title` 的 `top` 改为 `40`（避开色块），并设置 `textStyle: { color: '#fff' }` |
| **统计框位置偏下 & 重叠风险** | `graphic.group.left: '65%', bottom: '18%'` 为相对定位，易受容器尺寸影响 | 百分比定位在不同分辨率下不稳定；且未考虑 `grid.bottom`（当前为 `'15%'`）的挤压 | ✅ 改为绝对定位：`left: 400, bottom: 40`（假设容器宽≥800px）<br>✅ **关键**：在 `grid` 中设置 `bottom: '25%'`（预留统计框空间），避免与 X 轴刻度重叠 |
| **字体未加粗** | `nameTextStyle.fontWeight: 'bold'` 已设，但 `axisLabel` 未设 | `axisLabel` 的 `fontStyle` / `fontWeight` 未配置，导致刻度数字不够醒目 | ✅ 在 `xAxis.axisLabel` 和 `yAxis.axisLabel` 中均添加：<br>`fontStyle: 'normal', fontWeight: 'bold', fontSize: 14` |
| **辅助线（X=0/Y=0）不纯净** | `markLine` 添加在 `series[0]`，但 ECharts 中 `markLine` 属于系列，**可能因系列渲染顺序导致层级异常**；且 `symbol: 'none'` 不能完全消除端点标记 | `markLine` 应作为全局辅助线，推荐用 `xAxis/yAxis.splitLine` 配合 `lineStyle` 控制 | ✅ **删除 `series[0].markLine`**<br>✅ 在 `xAxis.splitLine` 中添加：<br>`{ show: true, lineStyle: { color: '#FFA07A', type: 'dashed', width: 1 } }`，并设置 `xAxis.splitNumber: 0`（仅保留指定刻度线）<br>✅ 在 `yAxis.splitLine` 中同理，并通过 `yAxis.splitArea.show: false` 关闭区域填充 |
